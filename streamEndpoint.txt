## ✅ Solução Melhor: Proxy no Backend

A forma **mais confiável** é criar um endpoint de proxy que valida o token uma vez e depois mantém o stream:

### Backend - Adicionar Session/Cookie

```python
from fastapi import Cookie, Response
import secrets

# Dicionário para armazenar sessões válidas (use Redis em produção)
valid_sessions = {}

@app.post('/auth/start-stream', summary="Autenticar e iniciar sessão de stream")
async def start_stream_session(authorization: str = Header(None), response: Response = None):
    # Valida token
    if not authorization or not authorization.startswith("Bearer "):
        raise HTTPException(status_code=401, detail="Token inválido")
    
    token = authorization.replace("Bearer ", "")
    if not validate_token(token):
        raise HTTPException(status_code=401, detail="Token inválido")
    
    # Cria sessão temporária
    session_id = secrets.token_urlsafe(32)
    valid_sessions[session_id] = True
    
    # Define cookie de sessão
    response.set_cookie(key="stream_session", value=session_id, httponly=True, max_age=3600)
    
    return {"status": "authorized", "session_id": session_id}


@app.get('/rgb', summary="RGB Stream")
async def rgb_feed(stream_session: str = Cookie(None)):
    # Verifica se tem sessão válida
    if not stream_session or stream_session not in valid_sessions:
        raise HTTPException(status_code=401, detail="Sessão inválida")
    
    return StreamingResponse(generateRGB_Stream(), media_type='multipart/x-mixed-replace; boundary=frame')
```

# Explicação Detalhada: Proxy no Backend com Sessões

Vou explicar passo a passo como funciona este método de autenticação para streams.

## 🎯 O Problema Inicial

Quando você faz:
```javascript
img.src = "http://10.0.30.175:8000/rgb";
```

O navegador faz uma requisição HTTP GET **diretamente** para o endpoint `/rgb`. O problema é que a tag `<img>` **não permite adicionar headers personalizados** (como `Authorization: Bearer token`).

## 💡 A Solução: Autenticação em Dois Passos

### **Passo 1: Autenticação Prévia (cria uma "chave temporária")**
### **Passo 2: Stream usa a "chave temporária" (cookie)**

---

## 📋 Como Funciona Detalhadamente

### **1. Cliente Pede Permissão para Ver o Stream**

```javascript
// Frontend envia o token para o servidor
const authResponse = await fetch('http://10.0.30.175:8000/auth/start-stream', {
    method: 'POST',
    headers: {
        'Authorization': `Bearer meu_token_secreto_12345`  // Token vai no header
    },
    credentials: 'include'  // Importante! Permite receber e enviar cookies
});
```

**O que acontece:**
- O JavaScript envia o token de autenticação no **header HTTP**
- O token **NÃO aparece no URL**
- `credentials: 'include'` diz ao navegador para aceitar cookies na resposta

---

### **2. Servidor Valida o Token e Cria uma Sessão**

```python
@app.post('/auth/start-stream')
async def start_stream_session(authorization: str = Header(None), response: Response = None):
    # 1. Extrai o token do header
    token = authorization.replace("Bearer ", "")
    # token agora é: "meu_token_secreto_12345"
    
    # 2. Valida se o token é válido
    if not validate_token(token):
        raise HTTPException(status_code=401, detail="Token inválido")
    
    # 3. Cria um ID de sessão único (como um "passe temporário")
    session_id = secrets.token_urlsafe(32)
    # session_id = "8kF3nQ7mP2xR9vL4wS6tY1jH5dC0bN"
    
    # 4. Guarda essa sessão como válida
    valid_sessions[session_id] = True
    # valid_sessions = {
    #     "8kF3nQ7mP2xR9vL4wS6tY1jH5dC0bN": True
    # }
    
    # 5. Envia um COOKIE para o navegador com essa sessão
    response.set_cookie(
        key="stream_session",           # Nome do cookie
        value=session_id,                # Valor: o ID único gerado
        httponly=True,                   # JavaScript não pode ler (mais seguro)
        max_age=3600                     # Válido por 1 hora
    )
    
    return {"status": "authorized"}
```

**O que acontece:**
- ✅ Token validado → Usuário autenticado
- 🎫 Servidor cria um **"passe temporário"** (session_id)
- 🍪 Servidor envia um **cookie** para o navegador com esse passe
- 💾 Servidor guarda que esse passe é válido (em memória ou banco de dados)

---

### **3. Navegador Recebe o Cookie Automaticamente**

Quando a resposta HTTP chega, o navegador vê:

```
HTTP/1.1 200 OK
Set-Cookie: stream_session=8kF3nQ7mP2xR9vL4wS6tY1jH5dC0bN; HttpOnly; Max-Age=3600
```

**O navegador automaticamente:**
- 🍪 Armazena o cookie `stream_session`
- 📤 Envia esse cookie em **TODAS as próximas requisições** para `10.0.30.175:8000`

---

### **4. Cliente Carrega a Imagem do Stream**

```javascript
// Agora o navegador já tem o cookie guardado
imgElement.src = "http://10.0.30.175:8000/rgb";
```

**O que acontece:**
- O navegador faz GET para `/rgb`
- **Automaticamente inclui o cookie** na requisição:
  ```
  GET /rgb HTTP/1.1
  Host: 10.0.30.175:8000
  Cookie: stream_session=8kF3nQ7mP2xR9vL4wS6tY1jH5dC0bN
  ```

---

### **5. Servidor Verifica o Cookie e Libera o Stream**

```python
@app.get('/rgb')
async def rgb_feed(stream_session: str = Cookie(None)):
    # 1. FastAPI extrai automaticamente o cookie
    # stream_session = "8kF3nQ7mP2xR9vL4wS6tY1jH5dC0bN"
    
    # 2. Verifica se esse "passe" é válido
    if stream_session not in valid_sessions:
        raise HTTPException(status_code=401, detail="Sessão inválida")
    
    # 3. Passe válido! Libera o stream
    return StreamingResponse(generateRGB_Stream(), media_type='multipart/x-mixed-replace; boundary=frame')
```

**O que acontece:**
- ✅ Cookie existe e é válido → Stream liberado
- ❌ Cookie inválido/ausente → Erro 401
- 📹 Stream CONTÍNUO funciona normalmente (MJPEG multipart)

## 🛡️ Melhorias de Segurança (Produção)

```python
import redis
from datetime import timedelta

# Use Redis ao invés de dicionário em memória
redis_client = redis.Redis(host='localhost', port=6379, db=0)

@app.post('/auth/start-stream')
async def start_stream_session(authorization: str = Header(None), response: Response = None):
    token = authorization.replace("Bearer ", "")
    
    # Valida JWT ou token
    if not validate_token(token):
        raise HTTPException(status_code=401)
    
    session_id = secrets.token_urlsafe(32)
    
    # Armazena no Redis com expiração automática
    redis_client.setex(
        f"stream_session:{session_id}",
        timedelta(hours=1),
        "valid"
    )
    
    response.set_cookie(
        key="stream_session",
        value=session_id,
        httponly=True,
        secure=True,      # Apenas HTTPS
        samesite="strict", # Proteção CSRF
        max_age=3600
    )
    
    return {"status": "authorized"}

@app.get('/rgb')
async def rgb_feed(stream_session: str = Cookie(None)):
    # Verifica no Redis
    if not redis_client.exists(f"stream_session:{stream_session}"):
        raise HTTPException(status_code=401)
    
    return StreamingResponse(generateRGB_Stream(), 
                           media_type='multipart/x-mixed-replace; boundary=frame')
```

## ❓ Perguntas Frequentes

**P: O cookie é seguro?**  
R: Sim! Com `httponly=True`, JavaScript malicioso não pode lê-lo. Use `secure=True` em HTTPS.

**P: E se o cookie expirar durante o stream?**  
R: O stream para. Você pode renovar automaticamente antes de expirar.

**P: Posso ter múltiplos streams simultâneos?**  
R: Sim! Cada sessão é independente.
